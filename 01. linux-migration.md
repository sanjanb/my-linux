# ROG Strix G614JU Migration Guide

> **Target:** Windows 11 $\rightarrow$ CachyOS (Arch-based) + Hyprland
> **Hardware:** 13th Gen i7-13650HX | NVIDIA RTX 40-Series | 16GB RAM
> **Focus:** AI/ML Research & High-Performance Content Creation


## 🛠️ Phase 1: Pre-Migration (Windows Side)

Before touching the partitions, prepare the hardware for a Linux kernel.

* [ ] **BitLocker:** Disable in `Settings > Device Encryption`.
* [ ] **Fast Startup:** Disable in `Power Options` (prevents SSD locking).
* [ ] **BIOS Config:** - Disable **Secure Boot**.
* Set SATA to **AHCI**.


* [ ] **Partitioning:** Shrink `C:` by **100GB+** using *Disk Management* to create "Unallocated Space."
* [ ] **Backup:** Move `.ssh`, `.gitconfig`, `.env`, and browser bookmarks to external storage.


## 💿 Phase 2: Installation

Use the **CachyOS** installer for its x86-64-v3/v4 optimized binaries.

1. **Flash ISO:** Use **Ventoy**; copy the CachyOS ISO to the drive.
2. **Boot:** Select the **NVIDIA** boot option.
3. **Installer Choices:**
* **Partitioning:** Manual -> Select Unallocated Space.
* `/boot/efi` (FAT32, 1GB).
* `/` (BTRFS, remaining space).


* **Desktop:** Hyprland.
* **Kernel:** `linux-cachyos` (Default).



## ⚡ Phase 3: Hardware Optimization

Post-install commands to handle the ROG-specific components.

```bash
# 1. Update and install ROG tools
sudo pacman -Syu
sudo pacman -S asusctl supergfxctl rog-control-center
sudo systemctl enable --now supergfxd

# 2. Battery & GPU Management

supergfxctl -m Hybrid                 # Set MUX switch to Hybrid mode

```

### 1. The "Internship Pro" Habit: Snapshotting

In the AI world, you will constantly be installing experimental libraries (`torch`, `cuda`, `tensorrt`). Sometimes they conflict.

* **The Suggestion:** Before you run a major `pacman -Syu` or a `pip install` that looks risky, take a **BTRFS Snapshot**.
* **Why:** If the system breaks, you can reboot, select the snapshot from the GRUB menu, and be back at work in 30 seconds. Your mentor will be impressed by your uptime, even when you're experimenting.

### 2. Monitor Hygiene: The "Hertz" Trap

Your ROG screen is high-refresh-rate (165Hz or 240Hz).

* **The Suggestion:** In your Hyprland config, ensure your monitor is actually set to its maximum refresh rate. Many Linux users accidentally leave it at 60Hz.
* `monitor=eDP-1, 2560x1600@240, 0x0, 1`


* **The Benefit:** High refresh rates reduce eye strain during long coding sessions. However, **when on battery**, use a keybind to drop it to 60Hz to gain an extra hour of life.

### 3. Peripheral Strategy: The "Home Station"

Your laptop has a **Thunderbolt 4** port and an **HDMI 2.1** port.

* **The Suggestion:** If you plan on doing serious content creation or deep AI work, get a **Laptop Cooling Pad** (like the *IETS GT500* or *Flydigi*).
* **Why:** These pads create a pressure seal. For a laptop like the G614JU which pulls massive wattage, active cooling can keep your clock speeds higher for longer during video rendering or model training.

### 4. Software "Garbage Collection"

Linux doesn't slow down over time like Windows, but it does collect "Orphaned" packages.

* **The Suggestion:** Once a month, run:
`sudo pacman -Rs $(pacman -Qdtq)`
* **Why:** This keeps your root partition lean and prevents your BTRFS snapshots from becoming unnecessarily large.

### 5. Protect your "Focus" (Workspace Logic)

Since you are using Hyprland, adopt a "Fixed Workspace" philosophy:

* **Workspace 1:** Communication (Slack/Discord/Mail)
* **Workspace 2:** Coding (VS Code/Cursor)
* **Workspace 3:** Terminal (AI Training/Logs)
* **Workspace 4:** Research (Browser)
* **Workspace 5:** Content Creation (OBS/Resolve)

### Final Maintenance Checklist (Every 6 Months)

* [ ] **Check the "Third Fan":** The ROG Strix G614 has a small fan in the center. It collects dust faster than the two big ones. Clean it!
* [ ] **Battery Report:** Run `upower -i /org/freedesktop/UPower/devices/battery_BAT0` to check your battery health percentage.
* [ ] **Clear Cache:** Run `paccache -r` to delete old versions of downloaded packages.
